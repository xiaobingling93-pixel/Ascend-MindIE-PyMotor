# 管理和监控接口

## 1. 启动探针接口

- 接口功能：供探针查询服务启动状态。

- 请求类型：GET

- URL：`http(s)://{CoordinatorIP}:{管理端口}/startup`

  > **说明**
  >
  > `{CoordinatorIP}`：Coordinator 服务部署机器的 IP 或域名，取值来自配置 `http_config.coordinator_api_host`（默认 `127.0.0.1`），参考 `deployer/user_config.json` 取值或实际运行时节点IP；`{管理端口}`：配置项 `http_config.coordinator_api_mgmt_port`（默认 `1026`）

- 请求参数：无

- 调用示例：

```bash
curl -X GET "http://{CoordinatorIP}:{管理端口}/startup"
```
- 响应示例：

```JSON
{ "status": "ok", "message": "Coordinator is starting up" }
```

## 5. 实例指标查询接口

- 接口功能：返回实例与端点维度的指标数据。
- 请求类型：GET
- URL：`http(s)://{CoordinatorIP}:{管理端口}/instance/metrics`

  > **说明**
  >
  > `{CoordinatorIP}`：Coordinator 服务部署机器的 IP 或域名，取值来自配置 `http_config.coordinator_api_host`（默认 `127.0.0.1`），参考 `deployer/user_config.json` 取值或实际运行时节点IP；`{管理端口}`：配置项 `http_config.coordinator_api_mgmt_port`（默认 `1026`）
- 请求参数：无
- 响应结构说明：
  - 指标条目字段：
    - `name`：指标名称。
    - `help`：指标说明。
    - `type`：指标类型（`counter` / `gauge` / `histogram` / `summary`）。
    - `label`：标签数组，元素为 Prometheus 标签字符串（如 `metric{label="value"}`）。
    - `value`：与 `label` 一一对应的数值数组。
  - `histogram` / `summary` 类型会包含 `*_bucket`、`*_count`、`*_sum` 等 label 项。
  - 指标较多时仅返回结构与部分示例，具体标签维度以实际运行时为准。
- 调用示例：

```bash
curl -X GET "http://{CoordinatorIP}:{管理端口}/instance/metrics"
```
- 响应示例：

```JSON
{
  "1": [
    {
      "name": "process_resident_memory_bytes",
      "help": "Resident memory size in bytes.",
      "type": "gauge",
      "label": ["process_resident_memory_bytes"],
      "value": [3683950592.0]
    },
    {
      "name": "http_requests_total",
      "help": "Total number of requests by method, status and handler.",
      "type": "counter",
      "label": [
        "http_requests_total{handler=\"/v1/chat/completions\",method=\"POST\",status=\"2xx\"}",
        "http_requests_total{handler=\"/v1/completions\",method=\"POST\",status=\"2xx\"}"
      ],
      "value": [101.0, 1.0]
    }
  ],
  "2": [
    {
      "name": "python_gc_objects_collected_total",
      "help": "Objects collected during gc",
      "type": "counter",
      "label": [
        "python_gc_objects_collected_total{generation=\"0\"}",
        "python_gc_objects_collected_total{generation=\"1\"}",
        "python_gc_objects_collected_total{generation=\"2\"}"
      ],
      "value": [68146.0, 10867.0, 2848.0]
    }
  ]
}
```

- 响应参数说明：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `<instance_id>` | object | 顶层键为实例 ID，值为该实例的指标列表。 |
| `<instance_id>[].name` | string | 指标名称。 |
| `<instance_id>[].help` | string | 指标说明。 |
| `<instance_id>[].type` | string | 指标类型。 |
| `<instance_id>[].label` | array | 标签数组，元素为 Prometheus 标签字符串。 |
| `<instance_id>[].value` | array | 与 `label` 一一对应的数值数组。 |

## 6. 实例刷新接口

- 接口功能：刷新 Coordinator 中的实例列表（add/del/set）。
- 请求类型：POST
- URL：`http(s)://{CoordinatorIP}:{管理端口}/instances/refresh`

  > **说明**
  >
  > `{CoordinatorIP}`：Coordinator 服务部署机器的 IP 或域名，取值来自配置 `http_config.coordinator_api_host`（默认 `127.0.0.1`），参考 `deployer/user_config.json` 取值或实际运行时节点IP；`{管理端口}`：配置项 `http_config.coordinator_api_mgmt_port`（默认 `1026`）
- 请求头：
  - 必选：`Content-Type: application/json`
  - 可选：无
- 请求参数：

| 参数名 | 类型 | 必选/可选 | 说明 |
|---|---|---|---|
| event | string | 必选 | 事件类型：`add` / `del` / `set` |
| instances | array | 必选 | 实例列表 |

- 调用示例：

```bash
curl -X POST "http://{CoordinatorIP}:{管理端口}/instances/refresh" \
  -H "Content-Type: application/json" \
  -d '{
    "event": "add",
    "instances": [
      {
        "job_name": "test-job",
        "model_name": "test-model",
        "id": 1,
        "role": "prefill",
        "endpoints": {
          "192.168.1.1": {
            "0": {
              "id": 0,
              "ip": "192.168.1.1",
              "business_port": "8080",
              "mgmt_port": "8081"
            }
          }
        }
      }
    ]
  }'
```

- 响应示例：

```JSON
{
  "request_id": "refresh_request",
  "status": "success",
  "message": "Instance refresh completed",
  "data": {
    "timestamp": "2026-01-29T12:00:00+00:00",
    "event_type": "add",
    "instance_count": 1
  }
}
```

- 响应参数说明：

| 参数名 | 类型 | 说明 |
|---|---|---|
| request_id | string | 请求标识 |
| status | string | 请求状态 |
| message | string | 响应消息 |
| data | object | 响应数据 |
| data.timestamp | string | 事件时间 |
| data.event_type | string | 事件类型，与请求 `event` 对应 |
| data.instance_count | integer | 实例数量 |

- 说明：请求体必须为 JSON，且大小不得超过 10MB。

## 7. MetaServer 转发接口

- 接口功能：仅在 PD/CDP 分离部署场景使用，用于 D 节点将请求转发至 P 节点。
- 请求类型：POST
- URL：`http(s)://{CoordinatorIP}:{管理端口}/v1/metaserver`

  > **说明**
  >
  > `{CoordinatorIP}`：Coordinator 服务部署机器的 IP 或域名，取值来自配置 `http_config.coordinator_api_host`（默认 `127.0.0.1`），参考 `deployer/user_config.json` 取值或实际运行时节点IP；`{管理端口}`：配置项 `http_config.coordinator_api_mgmt_port`（默认 `1026`）

- 请求头：
  - 必选：`Content-Type: application/json`
  - 可选：无
- 请求参数（JSON）：
  - 必选：OpenAI 兼容请求体（包含 `model` 与 `messages`/`prompt` 等字段）
  - 必选（转发时透传参数）：`kv_transfer_params`

| 参数 | 类型 | 必选 | 说明 |
| --- | --- | --- | --- |
| `model` | string | 是 | 模型名称，透传至目标节点。 |
| `messages` | array | 二选一 | Chat 输入，与 `prompt` 二选一。 |
| `prompt` | string | 二选一 | Completion 输入，与 `messages` 二选一。 |
| `stream` | boolean | 否 | 是否流式返回，透传至目标节点。 |
| `kv_transfer_params` | object | 是 | 转发控制参数。 |
| `kv_transfer_params.request_id` | string | 是 | 请求标识，用于跨节点跟踪与关联。 |
| `kv_transfer_params.do_remote_decode` | boolean | 否 | 是否在目标节点执行 Decode。 |
| `kv_transfer_params.do_remote_prefill` | boolean | 否 | 是否在目标节点执行 Prefill。 |
| `kv_transfer_params.remote_engine_id` | string | 是 | 目标节点引擎 ID。 |
| `kv_transfer_params.remote_host` | string | 是 | 目标节点地址（IP 或域名）。 |
| `kv_transfer_params.remote_port` | string | 是 | 目标节点端口。 |

- 调用示例（CDP 分离场景，D 节点触发 P 节点 Prefill）：

```bash
curl -X POST "http://{CoordinatorIP}:{管理端口}/v1/metaserver" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen3",
    "messages": [
      { "role": "user", "content": "Hello!" }
    ],
    "stream": false,
    "kv_transfer_params": {
      "request_id": "req-id",
      "do_remote_decode": false,
      "do_remote_prefill": true,
      "remote_engine_id": "engine-p-0",
      "remote_host": "10.0.0.12",
      "remote_port": "1000"
    }
  }'
```
- 响应示例（CDP 分离场景，透传 P 节点响应内容）：

```JSON
{
  "id": "chatcmpl-xxx12",
  "object": "chat.completion",
  "created": 1738828800,
  "model": "qwen3",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "Hello! How can I help you?"
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 6,
    "completion_tokens": 7,
    "total_tokens": 13
  }
}
```
- 调用示例（PD 分离场景，P 节点触发 D 节点 Decode）：

```bash
curl -X POST "http://{CoordinatorIP}:{管理端口}/v1/metaserver" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen3",
    "messages": [
      { "role": "user", "content": "Hello!" }
    ],
    "stream": false,
    "kv_transfer_params": {
      "request_id": "req-id",
      "do_remote_decode": true,
      "do_remote_prefill": false,
      "remote_engine_id": "engine-d-0",
      "remote_host": "10.0.0.21",
      "remote_port": "1001"
    }
  }'
```
- 响应示例（PD 分离场景，透传 D 节点响应内容）：

```JSON
{
  "id": "chatcmpl-xxx",
  "object": "chat.completion",
  "created": 1738828800,
  "model": "qwen3",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "Hello! How can I help you?"
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 8,
    "completion_tokens": 9,
    "total_tokens": 17
  }
}
```

- 响应参数说明（示例为非流式 `chat.completion`）：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `id` | string | 响应 ID。 |
| `object` | string | 响应对象类型，示例为 `chat.completion`。 |
| `created` | integer | 响应创建时间（Unix 时间戳）。 |
| `model` | string | 实际使用的模型名称。 |
| `choices` | array | 生成结果列表。 |
| `choices[].index` | integer | 结果序号。 |
| `choices[].message.role` | string | 角色，示例为 `assistant`。 |
| `choices[].message.content` | string | 生成内容。 |
| `choices[].finish_reason` | string | 结束原因，如 `stop`、`length` 等。 |
| `usage` | object | Token 统计信息。 |
| `usage.prompt_tokens` | integer | 输入 Token 数量。 |
| `usage.completion_tokens` | integer | 输出 Token 数量。 |
| `usage.total_tokens` | integer | 总 Token 数量。 |

## 8. 根路径服务信息接口

- 接口功能：返回 Coordinator 服务信息与接口索引。
- 请求类型：GET
- URL：`http(s)://{CoordinatorIP}:{管理端口}/`

  > **说明**
  >
  > `{CoordinatorIP}`：Coordinator 服务部署机器的 IP 或域名，取值来自配置 `http_config.coordinator_api_host`（默认 `127.0.0.1`），参考 `deployer/user_config.json` 取值或实际运行时节点IP；`{管理端口}`：配置项 `http_config.coordinator_api_mgmt_port`（默认 `1026`）
- 请求参数：无
- 调用示例：

```bash
curl -X GET "http://{CoordinatorIP}:{管理端口}/"
```

```JSON
{
```

- 响应参数说明：

| 参数 | 类型 | 说明 |
